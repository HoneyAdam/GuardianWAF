Features

Everything you need to protect your web applications. Nothing you don't.

🛡

Six Attack Detectors

SQL injection, XSS, path traversal, command injection, XXE, and SSRF. Tokenizer-based analysis with configurable scoring thresholds.

📦

Zero Dependencies

Pure Go, stdlib only. No CGO, no shared libraries, no supply chain risk. Single go build produces a static binary.

🚀

Three Deployment Modes

Standalone reverse proxy with dashboard, embeddable Go middleware, or lightweight sidecar for Docker/Kubernetes.

📊

Built-in Dashboard

Real-time monitoring, event viewer, rule management, and configuration — all embedded in the binary. No extra infrastructure.

🤖

MCP Server

15 structured tools for AI agent integration. Monitor, query, and manage your WAF through Claude Code or any MCP client.

⚡

Sub-millisecond Overhead

Full pipeline under 3µs per request. sync.Pool allocation, radix tree IP lookups, hand-written tokenizer — no regex on the hot path.

🔐

Bot Detection

JA3/JA4 TLS fingerprinting, User-Agent analysis, and behavioral tracking with sliding window. Detect scanners, headless browsers, and bots.

🚦

Rate Limiting

Token bucket algorithm with per-IP and per-path scoping. Configurable burst, auto-ban after repeated violations.

🔍

Response Protection

Security headers (HSTS, CSP, X-Frame-Options), credit card/SSN masking, API key redaction, stack trace stripping.

Quick Start

Pick your deployment mode and start protecting in seconds.

Standalone
Go Library
Sidecar
Dry-Run
# Install
go install github.com/guardianwaf/guardianwaf/cmd/guardianwaf@latest

# Run
guardianwaf serve -c guardianwaf.yaml
// go get github.com/guardianwaf/guardianwaf

waf, err := guardianwaf.New(guardianwaf.Config{
    Mode:      guardianwaf.ModeEnforce,
    Threshold: guardianwaf.ThresholdConfig{Block: 50, Log: 25},
})
if err != nil {
    log.Fatal(err)
}
defer waf.Close()

http.ListenAndServe(":8080", waf.Middleware(myHandler))
# Protect any backend with a single command
guardianwaf sidecar --upstream http://localhost:3000 --listen :8080

# Or with Docker
docker run -p 8080:8080 ghcr.io/guardianwaf/guardianwaf:latest \
  sidecar --upstream http://app:3000
# Test SQL injection detection
guardianwaf check --url "/search?q=' OR 1=1 --" -v
